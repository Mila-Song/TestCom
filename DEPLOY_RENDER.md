# Deploy `pdd` on Render (Free Tier)

## 1. Push Code to GitHub
- Put this project under a GitHub repo.
- Keep `pdd/render.yaml` in the repo.

## 2. Create Render Service
- In Render: `New` -> `Blueprint`.
- Connect your GitHub repo.
- Render reads `render.yaml` and creates the web service automatically.

## 3. Set Environment Variables
At minimum:
- `APP_SECRET_KEY` (auto-generated by `render.yaml` if not set manually)

Recommended:
- `PRIMARY_ADMIN_PASSWORD_HASH` (or `PRIMARY_ADMIN_PASSWORD`)
- `QWEN_API_KEY` (if you want AI generation/edit features)
- `QWEN_BASE_URL` (default in `.env.example`)
- `QWEN_IMAGE_MODEL` (optional)

You can copy from `.env.example`.

## 4. Start Command
Configured in `render.yaml`:

```bash
gunicorn app:app --bind 0.0.0.0:$PORT --workers 1 --threads 4 --timeout 120
```

## 5. Notes for Free Tier
- Free service may sleep after inactivity (cold start delay).
- Local file storage is not durable across rebuilds/redeploys.
  - This app currently stores uploaded assets in local disk.
  - For production durability, move assets/users metadata to object storage + database.
